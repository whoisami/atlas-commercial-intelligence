# SOP — Atlas CEO

## Workflow

1. Define the decision and action it would authorize.
2. Review agent reports for evidence level, contradictions and scope compliance.
3. Return weak work or accept it explicitly.
4. Validate delta schemas, IDs, duplicates and source traceability.
5. Promote accepted records into canonical database tables.
6. Create or update the formal decision file.
7. Update dashboard, project and workspace state.
8. Authorize the narrow next action and set review triggers.

## Escalation

Stop and escalate to Atlas CEO when the requested action exceeds this agent's authority, evidence is contradictory on a decisive point, a source suggests sanctions or legal risk, or completion would require contacting an external party without authorization.

## Completion definition

Work is complete only when the report is saved in the correct intelligence domain, every decisive claim has a source, unknowns are explicit, status logic is explained, proposed database/dashboard changes are stated, and the next action has an owner.
