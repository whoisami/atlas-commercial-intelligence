# Outputs — Learning Intelligence

## Learning report

Write `intelligence/learning/RXXX.md` using the next sequential report ID.

Required report sections:

1. Executive Learning Decision
2. Scope, Cohort and Evidence Cutoff
3. Evidence Ledger
4. Expected Versus Observed
5. Funnel and Speed Metrics
6. Lessons Register
7. Commercial Patterns
8. Messaging, Meeting, Negotiation and Pricing Findings
9. Alternative Explanations and Limitations
10. Playbook Change Decisions
11. Sources
12. Top Lessons
13. Immediate Improvements
14. Highest Commercial Opportunity
15. Highest Commercial Risk
16. Recommended Process Changes
17. Next Sprint Improvements
18. COMMERCIAL ACTION

The final block contains Decision, Confidence Score, Expected Business Impact, Knowledge Confidence and Next Improvement.

## Knowledge registers

Update, when evidence is relevant:

- `knowledge/lessons_learned.md`
- `knowledge/commercial_patterns.md`
- `knowledge/successful_outreach.md`
- `knowledge/failed_outreach.md`
- `knowledge/pricing_patterns.md`

Every entry retains a stable ID, evidence reference, evidence state, confidence, scope, date, commercial impact and recommended action.

## Playbooks

Continuously evaluate and, when the GO gate is met, update:

- `playbooks/email_playbook.md`
- `playbooks/meeting_playbook.md`
- `playbooks/negotiation_playbook.md`
- `playbooks/representation_playbook.md`
- `playbooks/pricing_playbook.md`
- `playbooks/commercial_principles.md`

Each adopted change requires a change-log entry with evidence, owner, effective date, review trigger and rollback condition. WATCH hypotheses remain experiments and do not silently replace the standard.

## Operational synchronization

Update:

- Learning-owned metrics in `dashboard/CEO_DASHBOARD.md`;
- the execution summary in `workspace/current_sprint.md`.

Dashboard values must cite the learning report and retain `UNKNOWN` when the cohort or denominator is unreliable.

## Decision meaning

- **GO:** implement the named learning/process improvement.
- **WATCH:** run the named bounded test or collect the named missing outcome.
- **NO-GO:** reject or retire the proposed lesson/process change.

No output authorizes external contact or a commercial commitment.
