# Checklist — Outreach Intelligence

## Start gate

- [ ] Manufacturer Due Diligence is GO.
- [ ] Manufacturer and contact IDs are stable.
- [ ] Recipient or department is verified.
- [ ] Contact channel and source are verified.
- [ ] Turkey-channel conflict is cleared or bounded.
- [ ] Prior outreach and do-not-contact status are checked.
- [ ] CEO/human authorization scope is explicit.
- [ ] Product, buyer and competition evidence is available.

## Evidence classification

- [ ] Important statements are VERIFIED, ESTIMATED or UNKNOWN.
- [ ] External copy uses only supported claims.
- [ ] Buyer claims use accepted Buyer Intelligence.
- [ ] ROI/price claims use accepted Pricing Intelligence.
- [ ] Turkey/channel claims use accepted Competition and Diligence evidence.
- [ ] Every UNKNOWN states impact and next verification.
- [ ] No facts, buyers, prices, results or relationships were invented.

## Manufacturer diagnosis

- [ ] Why the manufacturer should talk to Atlas is specific.
- [ ] Why Türkiye is relevant is evidenced.
- [ ] Why now is evidenced or UNKNOWN.
- [ ] Likely entry pain is a labeled hypothesis.
- [ ] Likely risk perception is a labeled hypothesis.
- [ ] Likely motivation is a labeled hypothesis.
- [ ] Traditional-distributor risk is explained without disparagement.
- [ ] Pilot outcome is measurable and realistic.
- [ ] Likely rejection reason is addressed.

## Customer sales strategy

- [ ] Selected buyer segments are evidenced.
- [ ] Product need and buyer problem are specific.
- [ ] Primary value driver is evidenced.
- [ ] Price is not assumed to be primary.
- [ ] Delivery, maintenance, downtime, equivalence, energy, spares and supply security were considered.
- [ ] Technical proof required is identified.
- [ ] Unsupported customer claims are excluded.

## Communication assets

- [ ] First email is personalized.
- [ ] LinkedIn connection request is personalized.
- [ ] LinkedIn first message is personalized.
- [ ] Follow-up #1 adds new value.
- [ ] Follow-up #2 adds new value.
- [ ] Follow-up #3 adds new value and a stop/close option.
- [ ] Phone script is role-appropriate.
- [ ] Meeting request is specific and low-friction.
- [ ] Meeting agenda is commercially useful.
- [ ] Assets do not repeat generic copy across channels.
- [ ] First message does not request exclusivity.
- [ ] Tone is commercial partner, not job applicant.

## Value proposition and pilot

- [ ] Manufacturer value proposition is company-specific.
- [ ] Risk-reduction message is clear.
- [ ] ROI/business-impact angle is evidenced or labeled.
- [ ] 90-day pilot has scope, activities, outputs and metrics.
- [ ] Manufacturer inputs are explicit.
- [ ] Brand/confidentiality controls are included.
- [ ] Exclusions and exit criteria are explicit.
- [ ] Sales are not guaranteed.
- [ ] Pilot terms are not presented as agreed.

## Objection library

- [ ] We already have a distributor.
- [ ] Turkey is not a priority.
- [ ] No budget.
- [ ] Not interested.
- [ ] Contact us later.
- [ ] Send more information.
- [ ] We work only with distributors.
- [ ] We do not use agents.
- [ ] We already sell directly.
- [ ] We need guaranteed sales.
- [ ] We require stock.
- [ ] We are concerned about brand risk.
- [ ] Each response includes diagnosis, reframe, next step and stop condition.
- [ ] Clear rejection is respected.

## Channel, timing and CRM

- [ ] Company size, export maturity, product, stage and decision maker shape the plan.
- [ ] Country adaptation avoids stereotypes.
- [ ] Primary and secondary channels have distinct purposes.
- [ ] Timing basis is evidenced or ESTIMATED.
- [ ] Follow-up spacing is defined.
- [ ] Channel overload is avoided.
- [ ] CRM schedule includes owner, objective, status and response branch.
- [ ] Every sequence step has separate authorization status.
- [ ] Stop, pause and opt-out conditions are explicit.

## Probability and decision

- [ ] Expected Response Rate has a defensible range or is UNKNOWN.
- [ ] Expected Meeting Probability has a defensible range or is UNKNOWN.
- [ ] Baseline, adjustments and limitations are explained.
- [ ] Confidence Score is 0–100 and evidence-based.
- [ ] GO Confidence Score ≥ 75.
- [ ] GO has verified recipient, channel and manufacturer facts.
- [ ] GO has a complete personalized sequence.
- [ ] GO has no blocking issue.
- [ ] GO means ready for approval, not sent.

## Stop rule

- [ ] Sufficient evidence supports the communication decision.
- [ ] Remaining unknowns do not affect truth or channel appropriateness.
- [ ] More research is unlikely to materially improve meeting probability.
- [ ] Analysis stopped without perfectionism.

## Outputs

- [ ] Report uses the next `intelligence/outreach/RXXX.md`.
- [ ] Report ends with the exact COMMERCIAL ACTION section.
- [ ] `database/outreach.csv` matches the report.
- [ ] `workspace/active_project.md` matches decision and next gate.
- [ ] Only Outreach Intelligence dashboard values were updated.
- [ ] Stable IDs and prior history are preserved.
- [ ] Report ID and verification date appear in updated surfaces.
- [ ] No draft is marked sent.
- [ ] No external action occurred without authorization.

## Integrity

- [ ] No generic emails.
- [ ] No private information or inferred contacts.
- [ ] No overpromised sales.
- [ ] No unsupported buyer-access claim.
- [ ] No exclusivity request in the first message.
- [ ] No deceptive identity or claim.
