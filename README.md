# Atlas Commercial Intelligence OS

Atlas OS is a documentation-first commercial intelligence operating system for international B2B market development.

Its immediate mission is the first qualified manufacturer meeting and representation agreement. Its durable purpose is to identify profitable industrial products, credible manufacturers, Turkish buyers, pricing gaps and defensible market-entry opportunities.

Atlas OS is not software and does not autonomously contact external parties.

## Architecture

| Area | Purpose |
|---|---|
| [agents](agents/) | Ten specialist operating contracts, prompts, SOPs and KPIs |
| [database](database/) | Canonical product, manufacturer, buyer, pricing, contact, outreach, meeting and decision tables |
| [intelligence](intelligence/) | Sequential, evidence-backed agent reports and proposed CSV deltas |
| [projects](projects/) | CEO-owned commercial project charters |
| [workspace](workspace/) | Active project, current sprint and next actions |
| [dashboard](dashboard/CEO_DASHBOARD.md) | Single source of truth for portfolio status |
| [knowledge](knowledge/) | Approved lessons and commercial patterns |
| [playbooks](playbooks/) | Governance, verification, reporting and sprint standards |
| [templates](templates/) | Controlled report, decision, project and evidence templates |
| [decisions](decisions/) | Binding Atlas CEO decisions only |

Legacy Sprint 001/002 material in `data/`, `outreach/`, `reports/` and `prompts/` remains intact. New work should follow Atlas OS ownership and naming rules.

## Start here

1. Read [Atlas OS Governance](playbooks/ATLAS_OS_GOVERNANCE.md).
2. Open [Active Project](workspace/active_project.md) and [Current Sprint](workspace/current_sprint.md).
3. Use the assigned agent contract in `agents/`.
4. Write evidence only to the assigned `intelligence/` domain.
5. Let Atlas CEO validate and promote accepted changes into canonical surfaces.
