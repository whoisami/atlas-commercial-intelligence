# Sprint Operating Cycle

## Start

1. Atlas CEO defines one commercial question and measurable outcome.
2. CEO records scope in `workspace/current_sprint.md`.
3. CEO assigns agent deliverables, report IDs and evidence cutoff.
4. Agents confirm required inputs exist.

## Execute

1. Agents research within scope.
2. Agents save evidence-backed reports and deltas under their assigned `intelligence/` domain.
3. Agents surface blockers early and do not broaden scope without approval.

## Review

1. CEO checks evidence, contradictions, duplication and decision thresholds.
2. CEO accepts, rejects or returns each deliverable.
3. CEO promotes accepted deltas into canonical database tables.
4. CEO updates dashboard and decision files.

## Close

1. CEO records result and next actions.
2. Learning Intelligence extracts reusable patterns.
3. CEO approves knowledge changes.
4. Sprint is closed only when every action has an owner and due date or is explicitly dropped.
